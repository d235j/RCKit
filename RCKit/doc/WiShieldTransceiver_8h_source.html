00017 /// This is one of several types of Transceiver that RCRx can use to communicate with
00018 /// an RCOIP transmitter.
00019 /// It works with the WiShield library, and with the following hardware:
00020 /// - WiShield WiFi shield
00021 /// - Yellowjacket (an Arduino with built in WiFi module)
00022 ///
00023 /// The default WiFi configuration is:
00024 /// - IP Address: 169.254.1.100
00025 /// - Gateway and transmitter address: 169.254.1.1
00026 /// - SSID: RCArduino
00027 /// - WiFi security type: NONE (i.e. an open unencrypted, no password WiFi network)
00028 /// - WiFi mode: ad-hoc (ie direct node-to-node communications)
00029 /// You can change these default settings by editing WiShieldTransceiver.cpp. (I know thats ugly, but
00030 /// it is a consequence of the way the WiShield library is built).
00031 ///
00032 /// There is only permitted to be one instance of WiShieldTransceiver. This is because of the low level 
00033 /// C code that glues the WiShield code to WiShieldTransceiver, and the fact there can also be only one instance of 
00034 /// the WiShield object.

00036 {   
00037 public:
00038     /// Constructor.
00039     /// Creates a new WiShieldTransceiver object.
00040     WiShieldTransceiver();

00042     /// Initialise the WiShield using the WiFi configuration in WiShieldTransceiver.cpp
00043     /// Call this once before using the Transceiver
00044     virtual void init();

00046     /// Poll the object for activity.
00047     /// This is expected to be called frequently in the main loop.
00048     /// It processes the WiFi stack, checking for received messages.
